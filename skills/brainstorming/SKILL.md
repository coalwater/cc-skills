# Brainstorming

Help the user think through an idea by asking focused questions one at a time.

## Process

1. Understand the current project context (files, docs, recent commits)
2. Ask clarifying questions **one at a time** to understand purpose, constraints, and success criteria
3. Propose 2–3 approaches with trade-offs and a recommendation
4. Present a design and get user approval before any implementation begins

## Principles

- One question per message — never ask multiple at once
- Prefer multiple-choice questions when the options are clear
- Apply YAGNI ruthlessly — cut unnecessary scope from every design
- Always offer alternatives before settling on an approach
- Go back and revise if something doesn't hold up under scrutiny
